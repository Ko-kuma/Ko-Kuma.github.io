PC방 홍보 문구나 고사양 게임의 사양표에서 흔히 보이는 클럭, 코어, 스레드는 CPU가 데이터를 처리하는 속도와 양을 결정짓는 핵심 설계 지표입니다.

## 1. 클럭 (Clock)

컴퓨터의 모든 부품은 클럭 신호라는 일정한 박자에 맞춰 움직입니다. 클럭 속도가 높아질수록 CPU는 동일한 시간 내에 더 많은 명령어 사이클을 처리할 수 있어 전반적인 성능이 향상됩니다.

* **측정 단위 (Hz):** 초당 클럭 반복 횟수를 의미합니다. 현대 CPU는 보통 $10^9$을 의미하는 GHz(기가헤르츠) 단위를 사용하며, 이는 1초에 약 10억 번 이상의 박자가 친다는 뜻입니다.
* **클럭의 가변성:** CPU는 발열 제어와 전력 효율을 위해 항상 일정하게 돌지 않습니다. 평상시의 **기본 클럭**과 작업량이 많을 때 일시적으로 속도를 높이는 **최대 클럭(부스트 클럭)**으로 나뉩니다.
* **오버클러킹 (Overclocking):** 제조사가 보증하는 속도 이상의 클럭을 강제로 인가하여 성능을 끌어올리는 기법입니다.
* **한계와 발열:** 클럭 속도와 성능은 정비례하지만, 속도가 빨라질수록 소비 전력이 늘고 **발열 문제**가 심각해집니다. 일정 온도를 넘어서면 CPU 보호를 위해 강제로 성능을 낮추는 쓰로틀링(Throttling)이 발생하므로 무한정 높일 수 없습니다.
* **비유:** 육상 선수가 발을 빨리 움직일수록 결승점에 빨리 도착하지만, 너무 무리해서 뛰면 체온이 오르고 심장에 무리가 가서 결국 쓰러지게 되는 것과 같습니다.

## 2. 코어와 멀티코어 (Core & Multi-core)

과거에는 클럭 속도를 높이는 데만 집중했으나 발열 장벽에 부딪히면서, CPU 내부에 명령어를 실행하는 실질적인 부품인 '코어'를 여러 개 넣는 **멀티코어(Multi-core)** 설계가 표준이 되었습니다.

### 멀티코어의 종류와 명칭
코어의 개수가 늘어날수록 동시에 처리할 수 있는 명령어의 양이 늘어납니다.

| 코어 개수 | 명칭 | 특징 및 용도 |
| :--- | :--- | :--- |
| **1개** | **싱글코어** | 한 번에 하나의 명령어만 처리 가능한 초창기 구조. |
| **2개** | **듀얼코어** | 다중 작업의 시작점. 일반적인 사무 업무에 적합. |
| **4개** | **쿼드코어** | 현대 컴퓨팅의 표준. 멀티태스킹의 기본 사양. |
| **6개** | **헥사코어** | 게이밍 및 고해상도 영상 시청에 유리. |
| **8개** | **옥타코어** | 고사양 게임 및 전문 작업용 하이엔드 사양. |
| **10개** | **데카코어** | 전문적인 워크스테이션 및 서버급 성능 제공. |
| **12개** | **도데카코어** | 4K 영상 편집, 3D 렌더링 등 초고사양 작업용. |

* **비유:** 클럭을 높이는 것이 '일꾼 한 명의 손을 빠르게 만드는 것'이라면, 코어를 늘리는 것은 '일꾼의 머릿수를 늘려 분업을 시키는 것'입니다.

## 3. 스레드와 멀티스레드 (Thread)

스레드는 '실행 흐름의 최소 단위'를 의미하며, 하드웨어에서 바라보는 관점과 소프트웨어(운영체제)에서 바라보는 관점이 다릅니다.

### ① 하드웨어적 스레드 (Hardware Thread)
하나의 코어가 동시에 처리할 수 있는 명령어의 가닥입니다. 최근 CPU는 하나의 물리 코어가 여러 작업을 동시에 잡을 수 있게 설계되는데, 이를 **멀티스레드 프로세서**라고 합니다.
* **하이퍼스레딩 (Hyper-Threading):** 인텔의 대표적인 기술로, 코어 내부의 유휴 자원을 활용해 하나의 물리 코어를 운영체제가 두 개의 코어처럼 인식하게 만드는 방식입니다.
* **논리 프로세서 (Logical Processor):** 운영체제(Windows 등)가 인식하는 CPU의 개수입니다. 예를 들어 6코어 12스레드 CPU라면, 운영체제는 12개의 CPU가 있다고 판단하여 작업을 분배합니다.

### ② 소프트웨어적 스레드 (Software Thread)
프로그램을 실행할 때 운영체제가 CPU에게 넘겨주는 작업의 단위입니다. 하나의 프로그램이 여러 개의 실행 흐름으로 나뉘어 동시에 돌아갈 때 이를 멀티스레드 프로그램이라고 합니다.

### 멀티스레드의 핵심 매커니즘
하드웨어적 멀티스레딩은 물리적인 코어는 하나지만, 마치 여러 명의 일꾼이 있는 것처럼 작업 효율을 극대화합니다.
* **비유:** 요리사(코어)는 한 명이지만, 숙련도가 높아 양손(스레드)을 자유자재로 사용하여 왼쪽 화구에서는 국을 끓이고 오른쪽 화구에서는 고기를 볶아 요리 시간을 단축하는 것과 같습니다.

## 요약
1. **클럭:** CPU의 심장 박동수. 빠를수록 좋지만 발열이 한계점.
2. **코어:** CPU 내부의 독립적인 실행 단위(일꾼). 12코어 이상의 다중 코어는 전문 작업에서 빛을 발함.
3. **스레드:** 운영체제가 보는 작업의 단위. 하이퍼스레딩을 통해 물리 코어 수보다 더 많은 작업을 동시에 수행 가능.
