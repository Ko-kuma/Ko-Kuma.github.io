프로그램 속 명령어와 데이터는 실행 전 후로 반드시 레지스터에 저장됩니다. 따라서, 레지스터 속 저장된 값만 관찰해도 프로그램의 실행 흐름을 파악할 수 있습니다. 다시 말해 레지스터 속 값을 유심히 관찰하면 프로그램을 실행할 때 CPU와 명령어의 흐름을 확인할 수 있습니다.

CPU 안에는 다양한 레지스터가 있고 각 레지스터는 각기 다른 역할을 가지고 있습니다. 이 레지스터의 이름과 역할을 학습합니다.

## 반드시 알아야 할 레지스터

**1. 프로그램 카운터 (Program Counter, PC)**
메모리에서 읽어 들일 **다음 명령어의 주소**를 저장하는 명령어 포인터이며, 
"다음은 100번지 명령어를 가져와야겠군" 하고 전체 흐름의 시작을 결정합니다.

**2. 명령어 레지스터 (Instruction Register, IR)**
방금 메모리에서 읽어 들인 **명령어 자체**를 저장하며, 
"가져온 명령어가 이거네, 내가 해석해볼게" 하고 제어장치가 읽을 수 있게 보관하고 해석을 기다리는 종착지입니다.

**3. 메모리 주소 레지스터 (Memory Address Register, MAR)**
CPU가 읽고자 하는 주소값을 주소 버스로 보내기 전에 거치는 **메모리의 주소** 저장소로, 
"100번지 주소 버스로 나갑니다. 비키세요" 하며 메모리로 주소를 쏴주는 통로가 됩니다.

**4. 메모리 버퍼 레지스터 (Memory Buffer Register, MBR)**
메모리와 주고받을 **값(데이터와 명령어)**을 저장하는 레지스터(MDR)로, 
"메모리에서 100번지 내용 받아왔습니다" 하며 실제 데이터를 담아오는 바구니 역할을 합니다.



**5. 플래그 레지스터 (Flag Register)**
ALU 연산 결과에 대한 부가 정보(양수/음수, 0 여부 등)나 CPU의 상태를 저장하여 프로그램 실행 중 필요한 '참고 정보'를 제공합니다.

**6. 범용 레지스터 (General Purpose Register)**
이름 그대로 다양하고 일반적인 상황에서 데이터와 주소를 모두 자유롭게 담을 수 있는 CPU의 다목적 서랍장입니다.

**7. 스택 포인터 (Stack Pointer)**
메모리 내 **스택 영역**의 최상단 주소를 가리켜 스택 주소 지정 방식에 사용되며, 이와 함께 아래와 같은 변위 주소 지정 방식들도 활용됩니다.
* **상대 주소 지정 방식**: 프로그램 카운터(PC) 값에 오퍼랜드(변위)를 더해 현재 위치 기준의 주소를 찾습니다.
* **베이스 레지스터 주소 지정 방식**: 베이스 레지스터 속 기준 주소에 오퍼랜드(변위)를 더해 주소를 찾습니다.

**8. 베이스 레지스터 (Base Register)**
주소 계산의 기준점이 되는 주소를 저장하며, 베이스 레지스터 주소 지정 방식에서 모든 계산의 **시작점** 역할을 수행합니다.

## 한눈에 보는 요약 정리

| 레지스터 | 핵심 기능 요약 |
| :--- | :--- |
| **PC** | "다음은 어디?" (다음 실행 주소) |
| **MAR** | "주소 출발!" (주소 전달 통로) |
| **MBR** | "데이터 도착!" (데이터 수령 바구니) |
| **IR** | "무슨 명령이지?" (명령어 해석 대기) |
| **SP** | "스택 최상단!" (스택 영역 관리) |
| **Base** | "여기서부터!" (주소 계산 기준점) |
