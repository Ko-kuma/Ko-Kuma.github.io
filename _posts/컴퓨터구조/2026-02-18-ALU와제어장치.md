## ALU와 제어장치

CPU의 구성 요소 중 ALU와 제어장치를 알아보고, ALU와 제어장치가 어떤 정보를 내보내고 받아들이는지를 중심으로 두 장치의 역할에 대해 학습합니다.

CPU는 메모리에 저장된 명령어를 읽어 들이고, 해석하고, 실행하는 장치입니다. 그리고 CPU 내부에는 계산을 담당하는 **ALU**, 명령어를 읽어 들이고 해석하는 **제어장치**, 그리고 정보를 저장하는 **레지스터**라는 구성 요소가 있습니다. 

이번에는 이 ALU와 제어장치에 대해 학습하겠습니다. ALU와 제어장치가 받아들이고 내보내는 정보를 기준으로 각 부품의 역할을 설명하고, 무엇을 내보내고 무엇을 받아들이는지 알아봅시다.

---

### 1. ALU (산술논리연산장치)

**ALU는 계산을 하는 부품입니다.** 계산을 하기 위해서는 피연산자와 수행할 연산이 필요합니다. (예: 1+2 같은 것)

* **ALU가 받아들이는 정보**
    * **피연산자**: 레지스터를 통해 피연산자를 받아들입니다.
    * **제어 신호**: 제어장치로부터 수행할 연산을 알려주는 제어 신호를 받아들입니다.
    * **정리**: 즉, ALU는 레지스터로부터 피연산자를 받아들이고 제어장치로부터 제어신호를 받아들여 산술 연산, 논리 연산 등 다양한 연산을 수행합니다.

* **ALU가 내보내는 정보**
    1.  **결과값**: 연산한 결과값은 특정 숫자나 문자, 메모리 주소가 될 수 있습니다.
        * **주의**: 이 결과값은 바로 메모리에 저장되지 않고 **일시적으로 레지스터에 저장**됩니다. 
        * **이유**: ALU의 결과값이 CPU를 거쳐 메모리에 직접 자주 접근하여 정보를 전송하게 되면, CPU의 정보 처리량에 과부하가 되기 때문입니다. 사람도 많은 정보를 한꺼번에 받아들이면 과부하가 되듯 컴퓨터도 마찬가지입니다. 그래서 처리를 위한 임시 기억장치(레지스터)에 저장한다는 개념입니다.
    2.  **플래그(Flag)**: 연산 결과에 대한 추가적인 상태 정보입니다. 
        * 이진수만 봐서는 음수인지 양수인지 판단하기 어렵기 때문에 구분을 위해 사용합니다. 가령 연산 결과가 음수일 때 ALU는 '방금 계산 결과가 음수'라는 추가 정보를 보냅니다. 혹은 연산 결과가 담길 레지스터보다 클 때(오버플로우), ALU는 '결과값이 크다'는 추가 정보를 내보냅니다.
        * 이러한 플래그들은 **플래그 레지스터**에 저장되어 CPU가 프로그램을 실행하는 도중 참고하는 중요한 정보가 됩니다.

#### [ALU가 보내는 대표적인 플래그]
| 플래그 종류 | 의미 | 사용 예시 |
| :--- | :--- | :--- |
| **부호 플래그** | 연산 결과의 부호를 나타냄 | 1이면 음수, 0이면 양수 (1인 것을 보고 음수임을 판단) |
| **제로 플래그** | 연산 결과가 0인지 여부 | 결과가 0이면 1, 아니면 0 |
| **캐리 플래그** | 올림수나 빌림수 발생 여부 | 연산 중 자리 올림이나 빌림이 발생했음을 알림 |
| **오버플로우 플래그** | 연산 결과가 표현 범위를 넘었는지 여부 | 결과값이 레지스터 용량보다 클 때 1 |
| **인터럽트 플래그** | 인터럽트 가능 여부 | 1이면 인터럽트를 허용, 0이면 무시 |
| **슈퍼바이저 플래그** | 커널/사용자 모드 확인 | OS 커널 모드 실행 중인지 확인하는 기준 |

### 2. 제어장치 (Control Unit)

**제어장치는 제어 신호를 내보내고 명령어를 해석하는 부품입니다.** 제어 신호는 컴퓨터 부품들을 관리하고 작동시키기 위한 일종의 전기 신호입니다. CPU 구성 요소 중 가장 정교한 부품이며, 제조사마다 차이가 있으나 가장 대표적인 정보는 다음과 같습니다.

* **제어장치가 받아들이는 정보**
    1.  **클럭(Clock) 신호**: 컴퓨터의 모든 부품을 일사불란하게 움직일 수 있게 하는 시간 단위입니다. 클럭 주기에 맞춰 레지스터 간 이동, ALU 연산, 명령어 읽기 등이 수행됩니다.
        * **오해 방지**: 모든 부품이 클럭에 의해 작동한다고 이해하면 안 됩니다. 클럭이라는 **박자**에 맞춰 컴퓨터 부품들이 작동할 뿐, 반드시 한 박자마다 매번 작동하는 것은 아닙니다.
    2.  **해석할 명령어**: CPU가 해석해야 할 명령어는 **명령어 레지스터(IR)**에 저장됩니다. 제어장치는 이 명령어를 받아들여 해석한 뒤 제어 신호를 발생시킵니다.
    3.  **플래그 값**: 플래그 레지스터 속 플래그 값을 받아들여 연산의 상태를 참고합니다.
    4.  **외부 제어 신호**: 시스템 버스 중 제어 버스로 전달된 외부의 제어 신호를 받아들입니다.

* **제어장치가 내보내는 정보 (제어 신호)**
    * **CPU 내부**: 레지스터 간의 데이터 이동을 지시하거나 ALU에게 연산을 수행하라고 명령합니다.
    * **CPU 외부**: 제어 버스를 통해 메모리에 접근(읽기/쓰기)하거나 입출력 장치를 제어합니다.
