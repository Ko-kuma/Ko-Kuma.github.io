명령어 파이프라이닝과 슈퍼 스칼라 기법을 실제로 CPU에 적용하려면 명령어가 파이프라이닝에 최적화되어 있어야 합니다. 쉽게 말해 CPU가 파이프라이닝과 슈퍼스칼라 기법을 효과적으로 사용하려면 **CPU가 인출하고 해석하고 실행하는 명령어가 파이프라이닝 하기 쉽게 생겨야 합니다.** 이와 관련한 CPU의 언어인 **ISA**와 각기 다른 성격의 ISA 기반인 **CISC**와 **RISC**를 정리해보겠습니다.

---

## 1. 명령어 집합(ISA, Instruction Set Architecture)

CPU는 제조사마다 규격과 기능, 만듦새가 다 다릅니다. 그래서 모든 CPU가 똑같은 명령어를 이해하는 것은 아닙니다. 이렇게 CPU가 직접 이해하고 실행할 수 있는 명령어들의 모음을 **명령어 집합** 또는 **명령어 집합 구조(ISA)**라고 합니다.

* **왜 '구조'라는 말이 붙나요?**
    * CPU가 어떤 명령어를 이해하는지에 따라 컴퓨터 내부의 설계 방식(레지스터 개수, 하드웨어 회로 등)이 완전히 달라지기 때문입니다.
* **하드웨어와 소프트웨어의 약속**
    * ISA는 CPU의 언어이자, 하드웨어가 소프트웨어를 어떻게 이해할지에 대한 약속입니다. 같은 프로그램이라도 ISA가 다르면 CPU는 그 말을 알아듣지 못합니다.
* **대표적인 종류**
    * **x86**: 인텔/AMD의 32비트용 언어.
    * **x86-64**: 현재 우리가 쓰는 대부분의 PC용 64비트 언어.

### [한눈에 보는 ISA 요약]
| 항목 | 설명 |
| :--- | :--- |
| **정의** | CPU가 이해할 수 있는 명령어들의 총집합 |
| **특징** | 제조사/규격마다 다르며, CPU 설계의 근본이 됨 |
| **비유** | 한국인에게는 한국어로, 미국인에게는 영어로 말해야 하는 것과 같음 |

## 2. CISC (Complex Instruction Set Computer)

CISC는 이름 그대로 **'복잡하고 다양한 명령어 집합을 사용하는 컴퓨터(CPU)'** 설계 방식입니다.

* **특징**: 다양하고 강력한 기능을 가진 명령어를 많이 사용합니다.
* **명령어의 형태**: 명령어의 크기와 모양이 제각각인 **가변 길이 명령어**를 사용합니다.
* **장점**: 
    * 하나의 명령어가 복잡한 일을 한꺼번에 처리해주기 때문에 프로그램을 실행하는 전체 명령어 수가 적습니다. 
    * 결과적으로 **컴파일된 실행 파일의 크기가 작아집니다.** (메모리가 귀했던 시절의 큰 장점입니다.)
* **단점**: 
    * 명령어 하나가 수행하는 시간이 제각각이라 **파이프라이닝**을 하기에 매우 어렵습니다. (언제 명령어가 끝날지 예측하기 힘들기 때문입니다.)

---

## 3. RISC (Reduced Instruction Set Computer)

RISC는 CISC와 정반대로 **'짧고 규격화된 단순한 명령어'**를 지향하는 방식입니다.

* **특징**: 되도록 **1클럭 내외**로 실행되는 단순한 명령어들로만 구성합니다.
* **명령어의 형태**: 모든 명령어의 크기가 똑같은 **고정 길이 명령어**를 사용합니다.
* **장점**: 
    * 명령어 모양이 예쁘게 규격화되어 있어 **명령어 파이프라이닝과 슈퍼스칼라** 기법을 적용하기에 압도적으로 유리합니다.
* **Load-Store 구조**: 
    * 메모리에 접근하는 명령어를 `Load`(가져오기)와 `Store`(저장하기)로 딱 제한합니다. 나머지 모든 연산은 속도가 가장 빠른 **레지스터** 안에서만 일어나게 해서 효율을 극대화합니다.

## 4. CISC와 RISC 핵심 비교 정리

| 비교 항목 | CISC (Complex) | RISC (Reduced) |
| :--- | :--- | :--- |
| **명령어 길이** | **가변 길이** (제각각임) | **고정 길이** (규격화됨) |
| **명령어 종류** | 많고 복잡함 | 적고 단순함 |
| **실행 클럭** | 여러 클럭 소모 (느림) | **대부분 1클럭** (빠름) |
| **주소 지정 방식** | 매우 다양함 | 단순함 (종류가 적음) |
| **메모리 접근** | 명령어 안에서 자유로움 | **Load / Store**로만 가능 |
| **파이프라이닝** | 구현이 어렵고 효율 낮음 | **구현이 쉽고 효율 높음** |
