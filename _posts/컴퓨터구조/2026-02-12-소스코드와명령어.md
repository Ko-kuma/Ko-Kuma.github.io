명령어에 대한 학습: 프로그램을 개발하기 위한 소스코드가 어떻게 컴퓨터를 작동하기 위한 명령어로 변환되는지 정리합니다.

## 1. 고급언어와 저급언어

우리가 작성하는 코드는 **고급언어**(개발자가 읽고 쓰기 편한 언어)이며, 컴퓨터가 이해하기 위해서는 **저급언어**(컴퓨터가 이해하고 실행하는 언어)로 변환되어야 합니다.

### 고급언어의 예시
* **C언어 및 Python**

``` bash

  #C
  include <stdio.h>
  int main(){
      printf("hello, world!");
      return 0;
  }
  
  #python
  
  print("hello, world!")


```



### 저급언어의 종류

저급언어는 **기계어**와 **어셈블리어**가 있습니다.

1. **기계어**: 이진수(0, 1)와 십육진수(0~9, A~F)로 이루어진 언어입니다.
2. **어셈블리어**: 기계어를 사람이 읽기 편한 형태로 번역한 언어입니다. 소스코드에 직접 이용되기도 합니다.
* **어셈블리어 예시**:

``` asm
push rbp
mov rbp, rsp
mov DWORD PTR [rbp-4], 1
mov DWORD PTR [rbp-8], 2

```

## 2. 고급언어에서 저급언어로의 변환 과정

변환 방식에는 크게 **컴파일**과 **인터프리트** 2가지가 있습니다.

### ① 컴파일 언어 (Compile Language)

* **방식**: 컴파일 언어 -> **컴파일** -> 저급언어
* **과정**: 고급언어(소스코드) -> **컴파일러** -> 저급언어(**목적코드**)
* 소스코드는 컴파일러에 의해 저급언어로 변환되는데, 이 과정을 **컴파일**이라고 합니다.
* 컴파일 결과로 저급언어인 **목적코드(Object Code)**가 생성됩니다.
* **C, C++** 등은 이 과정에서 **전처리**나 **어셈블** 같은 추가적인 기능도 포함됩니다.
* **오류 발생 시**: 소스코드 컴파일 중 오류가 발생하면 **소스코드 전체가 실행되지 않습니다.**

### ② 인터프리터 언어 (Interpreter Language)

* **방식**: 인터프리터 언어 -> **인터프리트** -> 저급언어
* **과정**: 인터프리터에 의해 **한 줄씩** 실행됩니다.
* 소스코드 전체가 저급언어로 변환되기까지 기다릴 필요가 없으며, 컴파일러에게 미리 부탁할 필요도 없습니다.
* **오류 발생 시**: 소스코드 인터프리트 중 오류가 발생하면, **오류 발생 전까지의 코드는 정상적으로 실행**됩니다.

## 비유

* **컴파일 방식**: 책 한 권을 통째로 번역해서 주는 것 (전체 변환)
* **인터프리터 방식**: 책 한 권의 내용을 한 줄씩 실시간으로 전달하는 것 (순차 실행)
