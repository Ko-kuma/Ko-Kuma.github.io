# 시작하기전 ..
코드가 길어지고 복잡해질수록 우리는 "어떻게 하면 이 코드를 효율적으로 관리할 수 있을까?"를 고민하게 됩니다. 그 해답이 바로 **함수(Function)**입니다. 믹서기에 과일을 넣으면 주스가 되어 나오듯이, 입력값을 넣으면 결과값을 돌려주는 마법상자, 함수에 대해 알아봅니다.



## 1. 함수란 무엇이며 왜 사용할까?

### 함수란?
어떤 입력값(Input)을 주었을 때, 내부에서 특정 작업을 수행하고 그 결과값(Output)을 돌려주는 '코드의 묶음'입니다.

### 사용하는 이유
1.  **재사용성:** 똑같은 내용을 반복해서 작성할 필요 없이, 필요할 때마다 호출해서 쓸 수 있습니다.
2.  **유지보수:** 코드를 수정할 때 함수 내부만 고치면 되므로 관리가 편합니다.
3.  **가독성:** 프로그램의 흐름을 한눈에 파악하기 쉽습니다.

## 2. 파이썬 함수의 기본 구조

```python
def 함수이름(매개변수):
    수행할_문장1
    수행할_문장2
    ...
    return 리턴값

```

* `def`: 함수를 만들겠다는 예약어(define).
* `함수이름`: 함수를 호출할 때 사용할 이름.
* `매개변수`: 함수 안에서 사용할 재료(입력값)를 받는 변수.
* `return`: 함수의 결과를 돌려주는 명령어.

## 3. 매개변수와 인수 (용어 주의!)

혼용해서 쓰기도 하지만, 엄밀히 말하면 차이가 있습니다.

* **매개변수 (Parameter):** 함수를 **정의할 때** 사용하는 변수 (껍데기)
* **인수 (Arguments):** 함수를 **호출할 때** 실제로 넣어주는 값 (알맹이)

```python
def add(a, b):  # a, b는 매개변수
    return a + b

print(add(3, 4)) # 3, 4는 인수

```

## 4. 입력값과 리턴값에 따른 함수의 4가지 형태

함수라고 해서 반드시 입력과 출력이 다 있어야 하는 것은 아닙니다.

### ① 일반적인 함수 (입력 O, 리턴 O)

> 가장 많이 쓰이는 형태입니다.

```python
def add(a, b):
    result = a + b
    return result

a = add(3, 4) # a에 7이 담김

```

### ② 입력값이 없는 함수 (입력 X, 리턴 O)

> 괄호 안이 비어있습니다.

```python
def say():
    return "Hi"

a = say() # a에 "Hi"가 담김

```

### ③ 리턴값이 없는 함수 (입력 O, 리턴 X)

> `return` 문이 없거나 `return`만 단독으로 쓰입니다. 결과값을 변수에 담을 수 없습니다. (담으면 None이 됨)

```python
def add(a, b):
    print(f"{a}와 {b}의 합은 {a+b}입니다.")

a = add(3, 4) # 출력은 되지만, a에는 None이 저장됨

```

### ④ 입력값도 리턴값도 없는 함수 (입력 X, 리턴 X)

```python
def say():
    print("Hi")

say()

```

## 5. 매개변수를 지정하여 호출하기

함수를 호출할 때 매개변수의 이름을 직접 지정하면 순서를 바꿔도 상관없습니다.

```python
def sub(a, b):
    return a - b

print(sub(a=7, b=3)) # 4
print(sub(b=3, a=7)) # 4 (순서가 바뀌어도 a와 b를 찾아감)

```

## 6. 입력값이 몇 개가 될지 모를 때 (`args`)

여러 개의 입력값을 한 번에 받고 싶을 때 사용합니다. 변수명 앞에 별표(`*`)를 붙입니다. 보통 `args`를 많이 씁니다.

```python
def add_many(*args): 
    result = 0 
    for i in args: 
        result = result + i 
    return result 

print(add_many(1, 2, 3))      # 6
print(add_many(1, 2, 3, 4, 5)) # 15

```

* `args`는 입력값들을 **튜플(tuple)** 형태로 만들어줍니다.

## 7. 키워드 매개변수 (`kwargs`)

매개변수 앞에 별 두 개(`**`)를 붙입니다. `key=value` 형태로 입력받으며, **딕셔너리(dict)** 형태로 저장됩니다.

```python
def print_kwargs(**kwargs):
    print(kwargs)

print_kwargs(name='foo', age=3)
# 결과: {'name': 'foo', 'age': 3}

```

## 8. 함수의 리턴값은 언제나 하나이다

함수는 `return`을 만나는 순간 결과를 반환하고 **종료**됩니다.

```python
def add_and_mul(a, b): 
    return a+b, a*b

result = add_and_mul(3, 4)
print(result) # (7, 12) <- 튜플 하나로 반환됨

```

* **팁:** `return`을 두 번 쓴다고 해서 두 번 실행되지 않습니다. 첫 번째 `return`에서 함수는 끝납니다.
* **return의 또 다른 쓰임새:** 반복문을 `break`로 빠져나오듯, 함수 중간에 즉시 빠져나오고 싶을 때 `return`을 단독으로 사용할 수 있습니다.

## 9. 매개변수에 초깃값 미리 설정하기

입력값을 주지 않았을 때 기본으로 사용할 값을 설정할 수 있습니다.
**주의:** 초기값이 설정된 매개변수는 **항상 뒤쪽에** 위치해야 합니다.

```python
def say_myself(name, old, man=True): # man=True가 기본값
    print(f"나의 이름은 {name} 입니다.")
    print(f"나이는 {old}살입니다.")
    if man:
        print("남자입니다.")
    else:
        print("여자입니다.")

say_myself("박응용", 27)          # man을 입력하지 않아도 True로 동작
say_myself("박응용", 27, False)   # man 자리에 False 입력 가능

```

## 10. 함수 안에서 선언한 변수의 효력 범위 (Scope)

함수 안에서 만든 변수(`지역 변수`)는 함수 밖에서 사용할 수 없습니다.

```python
a = 1
def vartest(a):
    a = a + 1 # 여기서 a는 함수 안에서만 쓰이는 임시 변수

vartest(a)
print(a) # 결과는 1 (함수 안의 a는 밖의 a에 영향을 주지 않음)

```

## 11. 함수 안에서 밖의 변수를 변경하려면?

### ① return 사용하기 (권장)

```python
a = 1 
def vartest(a): 
    a = a + 1 
    return a

a = vartest(a) # 리턴값을 다시 a에 대입
print(a) # 2

```

### ② global 명령어 사용하기 (비권장)

```python
a = 1 
def vartest(): 
    global a # 함수 밖의 변수 a를 직접 가져다 쓰겠다
    a = a + 1

vartest() 
print(a) # 2

```

* `global`을 사용하면 함수가 외부 변수에 종속되어 독립성이 떨어지므로 가급적 피하는 것이 좋습니다.

## 12. lambda (람다) 예약어

함수를 한 줄로 간결하게 만들 때 사용합니다. 보통 함수를 매개변수로 넘겨야 할 때 많이 씁니다.

* **구조:** `lambda 매개변수1, 매개변수2, ... : 표현식`

```python
# 일반 함수
def add(a, b):
    return a + b

# 람다 함수 (위와 동일한 기능)
add = lambda a, b: a + b

print(add(3, 4)) # 7

```
