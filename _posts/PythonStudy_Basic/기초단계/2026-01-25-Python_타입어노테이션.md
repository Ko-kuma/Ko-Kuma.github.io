# 시작하기 전에..
파이썬은 쉽고 빠릅니다. 변수에 타입을 지정하지 않아도 알아서 척척 계산해 주니까요.
하지만 코드가 길어지고, 협업하는 동료가 생기면 이 '자유로움'이 독이 되기도 합니다. 파이썬의 유연함에 **안정성**을 더해주는 **타입 어노테이션**에 대해 알아봅니다.

---

## 1. 동적 언어 vs 정적 언어

프로그래밍 언어는 변수의 자료형(Type)을 언제 결정하느냐에 따라 크게 두 가지로 나뉩니다.



### ① 정적 언어 (Static Typing)
* **대표 언어:** C, C++, Java, Go, Rust
* **특징:** 변수를 만들 때 **미리 타입을 정해야** 합니다.
* **컴파일 시점**에 타입을 검사하므로, 타입 에러가 있으면 실행조차 되지 않습니다.
* 예시 (Java): `int num = 10;` (숫자만 넣겠다고 선언)

### ② 동적 언어 (Dynamic Typing)
* **대표 언어:** Python, JavaScript, Ruby
* **특징:** 변수의 타입이 **실행 시점(Runtime)**에 결정됩니다.
* 변수에 숫자도 넣었다가, 문자열도 넣었다가 할 수 있습니다.
* 예시 (Python): `num = 10` (타입 선언 없음)

---

## 2. 동적 언어(파이썬)의 장단점

파이썬은 대표적인 동적 언어입니다.

| 장점 (Pros)                                        | 단점 (Cons)                                         |
| :------------------------------------------------- | :-------------------------------------------------- |
| **개발 속도가 빠르다.** (타입 고민 없이 코딩 가능) | **실행해 봐야 에러를 알 수 있다.** (Runtime Error)  |
| 코드가 간결하고 읽기 쉽다.                         | 코드가 길어지면 이 변수에 뭐가 들어있는지 헷갈린다. |
| 진입 장벽이 낮다.                                  | 협업 시 문서화가 없으면 코드 분석이 어렵다.         |

> **문제 상황 예시:**
> ```python
> def add(a, b):
>     return a + b
> 
> # 의도: 숫자 더하기 (10 + 20 = 30)
> # 실수: 문자열 더하기 ("10" + "20" = "1020") -> 에러는 안 나지만 결과가 엉망이 됨
> ```

---

## 3. 파이썬 타입 어노테이션 (Type Annotation)

파이썬 3.5부터 도입된 기능으로, **"이 변수에는 이런 타입이 들어올 거야"** 라고 힌트(Hint)를 주는 것입니다.

### ① 기본 문법
변수 뒤에 `: 타입`, 함수 뒤에 `-> 리턴타입`을 적습니다.

```python
# 변수 타입 힌트
name: str = "홍길동"
age: int = 30
is_student: bool = True

# 함수 타입 힌트 (입력값 타입 -> 리턴값 타입)
def add(a: int, b: int) -> int:
    return a + b
