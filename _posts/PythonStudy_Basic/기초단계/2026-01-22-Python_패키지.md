# 시작하기 전에..
모듈이 여러 개 모이면 관리가 힘들어집니다. 이럴 때 도트(`.`)를 사용하여 모듈을 계층적(디렉터리 구조)으로 관리할 수 있게 해주는 것이 바로 **패키지(Package)**입니다.

## 1. 패키지란? (What is a Package?)

파이썬 패키지는 **모듈들을 모아놓은 디렉터리(폴더)**입니다.
* **모듈:** 하나의 `.py` 파일
* **패키지:** 모듈이 들어있는 폴더의 모음



## 2. 패키지 만들기

가상의 게임 프로그램을 만든다고 가정하고, 아래와 같은 폴더 구조를 만들어 봅니다.

```text
game/
    __init__.py
    sound/
        __init__.py
        echo.py
        wav.py
    graphic/
        __init__.py
        screen.py
        render.py

```

* **`game`**: 루트 패키지
* **`sound`, `graphic`**: 서브 패키지
* **`.py` 파일들**: 모듈

**예시 코드 (`echo.py`)**

```python
# game/sound/echo.py
def echo_test():
    print("echo")

```

## 3. 패키지 안의 함수 실행하기

패키지 안의 모듈을 불러오는 방법은 크게 3가지가 있습니다. (경로가 중요합니다!)

1. **모듈 전체를 import 하기**
```python
import game.sound.echo
game.sound.echo.echo_test()

```


2. **모듈만 import 하기**
```python
from game.sound import echo
echo.echo_test()

```


3. **함수 직접 import 하기**
```python
from game.sound.echo import echo_test
echo_test()

```



## 4. `__init__.py`란 무엇인가?

`__init__.py` 파일은 **"이 디렉터리는 단순한 폴더가 아니라 파이썬 패키지입니다"** 라고 파이썬에게 알려주는 역할을 합니다.
*(참고: 파이썬 3.3부터는 이 파일이 없어도 패키지로 인식하지만, 하위 호환성과 초기화 설정을 위해 생성하는 것이 정석입니다.)*

### `__init__.py`의 주요 용도

#### ① 패키지 초기화 (Initialization)

패키지가 처음 `import` 될 때, `__init__.py`에 작성된 코드가 **자동으로 실행**됩니다.

```python
# game/__init__.py
print("Initializing game package...")

```

```python
>>> import game
Initializing game package...

```

#### ② 패키지 변수 및 함수 정의

패키지 레벨에서 공통으로 사용할 변수나 함수를 정의할 수 있습니다.

```python
# game/__init__.py
VERSION = 3.5

def print_version():
    print(f"Game Version: {VERSION}")

```

```python
import game
print(game.VERSION) # 3.5
game.print_version()

```

#### ③ 패키지 내 모듈을 미리 import 하기

사용자가 `import game`만 해도 `game.sound.echo`를 사용할 수 있게 편의성을 제공할 때 사용합니다.

```python
# game/__init__.py
from .sound import echo

```

```python
# 사용자는 굳이 import game.sound.echo를 안 해도 됨
import game
game.echo.echo_test()

```

## 5. `__all__`의 역할 (`import *`)

만약 `from game.sound import *` 처럼 별표(`*`)를 써서 모든 모듈을 불러오고 싶다면 어떻게 해야 할까요?
기본적으로 파이썬은 `*`를 쓰면 아무것도 불러오지 않습니다. **`__init__.py`에 `__all__` 변수를 설정**해야 합니다.

```python
# game/sound/__init__.py
__all__ = ['echo'] # *를 쓰면 echo 모듈만 import 하겠다는 뜻

```

```python
from game.sound import *
echo.echo_test() # 가능
# wav.wav_test() # 불가능 (all 리스트에 없으므로)

```

## 6. Relative 패키지 (상대 경로)

패키지 안의 모듈들끼리 서로를 불러올 때 사용합니다.
예를 들어 `graphic/render.py`에서 `sound/echo.py`를 쓰고 싶다면 어떻게 할까요?

* **`..` (부모 디렉터리)**
* **`.` (현재 디렉터리)**

```python
# game/graphic/render.py
from ..sound.echo import echo_test

def render_test():
    print("render")
    echo_test() # 다른 폴더의 함수 호출

```

> **주의:** Relative Import는 모듈을 직접 실행(`python render.py`) 할 때는 사용할 수 없습니다. 반드시 패키지 전체를 통해 실행될 때만 동작합니다.

---

### 요약 정리

1. **패키지**: 모듈을 담는 **디렉터리** (도트 `.`로 계층 관리).
2. **`__init__.py`**: 패키지를 **초기화**하고 설정을 관리하는 파일.
* 변수 정의, 모듈 미리 로드 등의 역할을 수행함.


3. **`__all__`**: `from ... import *` 사용 시 불러올 모듈 목록을 지정.
4. **Relative Path**: `..`(부모) 등의 기호로 내부 모듈끼리 참조.
