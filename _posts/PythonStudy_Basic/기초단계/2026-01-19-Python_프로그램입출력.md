# 시작하기 전에..
우리는 지금까지 `input()` 함수를 통해 프로그램 실행 도중에 사용자의 입력을 받았습니다. 
하지만 도스(DOS)나 리눅스 터미널에서 명령어를 칠 때처럼, **프로그램을 실행함과 동시에 값을 전달**하고 싶을 때는 어떻게 할까요? 이때 사용하는 것이 바로 **프로그램 입출력**입니다.

## 1. 프로그램 입출력이란?

프로그램 입출력이란 프로그램 실행 시 **명령행(Command Line)**에서 인수(Arguments)를 직접 전달하는 방식을 말합니다.

* **비교:**
    * `input()`: 프로그램 실행 $\rightarrow$ 멈춤 $\rightarrow$ 값 입력 $\rightarrow$ 다시 진행
    * **프로그램 입력**: `python test.py data` 처럼 실행하는 순간 `data`라는 값을 휙 던져줌

## 2. sys 모듈이란?

파이썬에는 **`sys`** 라는 아주 중요한 표준 라이브러리가 있습니다.
`sys` 모듈은 **파이썬 인터프리터가 제공하는 변수나 함수를 직접 제어**할 수 있게 해줍니다. 프로그램 입출력을 다루기 위해서는 이 모듈을 불러와야 합니다.

```python
import sys

```

## 3. sys 모듈 사용하기 (`sys.argv`)

프로그램 실행 시 전달된 인수들은 `sys.argv`라는 곳에 **리스트(List)** 형태로 저장됩니다.

### 예제 코드 (`test.py`)

먼저 에디터로 아래 코드를 작성하여 `test.py`로 저장합니다.

```python
# test.py
import sys

print(sys.argv)

```

### 실행 방법 (터미널/CMD)

이 프로그램은 IDLE의 Run 버튼이 아니라, 터미널(명령 프롬프트)에서 실행해야 의미가 있습니다.

```bash
C:\doit> python test.py abc 123

```

### 실행 결과

```python
['test.py', 'abc', '123']

```

**[분석]**
`sys.argv`는 입력받은 값들을 리스트로 만듭니다.

* `sys.argv[0]`: 파일 이름 (`test.py`)
* `sys.argv[1]`: 첫 번째 인수 (`abc`)
* `sys.argv[2]`: 두 번째 인수 (`123`)

> **주의:** 항상 **0번째 요소는 실행한 파일명**이 들어간다는 점을 기억해야 합니다! 우리가 실제로 전달한 데이터는 인덱스 1부터 시작합니다.

## 4. 쉽게 이해하기 (활용 예제)

"왜 굳이 이렇게 쓸까요?"
파일을 복사하거나 변환하는 프로그램을 만든다고 상상해 보세요.

**상황: 문자열을 모두 대문자로 바꿔주는 프로그램(`upper.py`) 만들기**

**방법 A (`input` 사용)**

1. 프로그램 실행
2. "변환할 단어를 입력하세요: " (기다림)
3. 사용자가 "hello" 입력
4. 결과: "HELLO"

**방법 B (`sys` 사용)**

* 명령어: `python upper.py hello`
* 결과: "HELLO" (기다림 없이 즉시 실행)

### 방법 B의 코드 (`upper.py`)

```python
import sys

# 입력받은 인수(hello)를 대문자로 변환하여 출력
# sys.argv[1]부터가 진짜 내용임
args = sys.argv[1:] 

for i in args:
    print(i.upper(), end=' ')

```

**실행 결과:**

```bash
C:\doit> python upper.py life is too short
LIFE IS TOO SHORT

```

---

**요약:**

1. **`import sys`** 를 통해 모듈을 가져온다.
2. **`sys.argv`** 를 사용하면 명령행에서 입력한 값들을 **리스트**로 받을 수 있다.
3. `sys.argv[0]`은 파일명이고, **`sys.argv[1]`부터**가 사용자가 입력한 진짜 데이터다.
